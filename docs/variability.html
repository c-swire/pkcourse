Definition of a covariate

In statistical terms a variable is something that can be quantified in some way - e.g. measured or counted.

A covariate is a variable that changes (varies) with (co) the outcome of interest. In modeling covariates are variables additional to the core predictor variable in the model (e.g. dose) that may be used to help model the outcome variable (e.g. concentration). An example might be that weight would be expected to affect volume of distribution for many drugs and so it might be reasonable to include a weight effect on the volume of distribution parameter in a pharmacokinetic model.

Covariates can be continuous, categorical, discrete…

Why bother to include covariates in a model?

We have discussed how non-linear mixed-effects models allow us to describe inter-individual differences in pharmacokinetic parameters - these are our \(\eta\) terms.

As an extension of this, it would be nice to try and understand and explain some of this variability so that we can:

  • Explain it biologically, rather than just mathematically
  • Use this information to make decisions about our drugs
    • For example if we find that renal function is linked with clearance of a drug, we might recommend different doses for those with kidney disease

Choosing covariates to include in a model

The choice of what covariates one should include in a model is complex.

Options include:

  • Test the effect of all variables in a data set on all parameters in the model. This can be a time-consuming (and processor hungry) process.
  • Knowledge about the drug’s pharmacology (prior information) may guide the choice of which variables to test. E.g. if a drug is renally excreted, you might choose to test the effect of some measure of kidney function on drug clearance.
  • There are some tools that can help us. For example we can plot pharmacokinetic parameters (or \(\eta\)) against potential covariates to look for trends/relationships to test

Introduction to model diagnostics

There are a wealth of diagnostic plots that can be used to think about how well a model fits data from an experiment. We will look at a few key examples.

These approaches can be used to assess whether adding a covariate improves a model’s fit.

You are likely to come across these plots and terms if you read any population pharmacokinetic modelling papers. We have explained them in a bit of detail so you should be able to interpret/understand these plots in future.

DV vs PRED and DV vs IPRED

The most straightforward diagnostic plot available is a plot of observations (dependent variable, DV) vs predicted concentrations from a model. There are two subsets of this type of plot. The first, DV vs PRED takes predictions based on population parameters (i.e. excluding \(\eta\)s) and the second is DV vs IPRED, which bases predictions on individual values for model parameters (including \(\eta\)s).

Figure 1 shows these diagnostic plots for the furosemide data we used in Session 2.

modfit<-data.frame(fit) # make a dataframe out of the output of the model

ggplot(modfit,aes(x=PRED,y=DV))+
  geom_point()+
  theme_bw(base_size=18)+
  geom_abline(intercept = 0, slope=1)+
  geom_smooth(col='red',se=F)

ggplot(modfit,aes(x=IPRED,y=DV))+
  geom_point()+
  theme_bw(base_size=18)+
  geom_abline(intercept = 0, slope=1)+
  geom_smooth(col='red',se=F)

Figure 1: Basic diagnostic plots for furosemide one-compartment, oral-absorption model fit from Chapter 4

Take a look at Figure 1(a) first. A perfect model fit would have predictions that were identical to the observations from the experiment. All of the points in the scatterplot would fall in a straight line along the line of unity (\(DV=PRED\)). However, real-world data are never perfect, so what we are really looking for in this plot is even spread of points above and below the line of unity. Any groups of data points deviating away from this line of unity might make us suspicious that there is some underlying variability not being modelled.

In our example in Figure 1(a) we can see that there is pretty reasonable fit, with even distribution of data points. A red ‘line of best fit’ tracks fairly evenly along the line of unity. This suggests our model fit is pretty good.

Figure 1(b) shows DV vs IPRED. This is individualised predictions, taking into account random effect associated with an individual (\(\eta\)). Again, the perfect model fit has all lines along the line of unity. DV vs IPRED plots will generally always look a little better than DV vs PRED plots in this regard. However, it is unusual to get such a tight fit as we see in Figure 1(b) as usually there is additional unexplained variability in data. The example here looks relatively perfect because I simulated these data and I did not simulate complex variability!

Conditionally-weighted residuals

When we fit a pharmacokinetic model to data, we can assess how well the model fits by examining the difference between each observed concentration and the concentration our model predicted. This difference is called a residual.

However, raw residuals can be misleading because their size tends to scale with the magnitude of the observation: residuals around a high concentration early after dosing will naturally be larger than those around a low concentration at later time points, making it difficult to assess model fit consistently across the full concentration-time profile.

Conditionally weighted residuals (CWRES) address this by standardising the residuals so that they are comparable across the full range of observations.

If the model fits well, CWRES should be roughly normally distributed with a mean of zero and a standard deviation of one, meaning we would expect approximately 99.7% of values to fall between −3 and +3.

When we plot CWRES against time, we are looking for an even, random scatter of points around zero. Any systematic trend, such as a U-shape or a consistent drift above or below zero, suggests that the model is not capturing something important in the data and may need to be refined.

nlmixr2 will output CWRES through an additional tableControl() command.

       list(print=0),

The Objective Function Value (OFV)

Objective function value is a measure of model fit. I do not propose that you learn in-depth how it is calculated or what its mathematics are.

What is important for you to note is that the objective function can be used to compare two models. In general, a lower objective function value indicates a superior model fit.

Visual Predictive Check (VPC)

The diagnostic plots we have used so far (DV vs PRED, DV vs IPRED, CWRES) assess model fit by examining residuals. A complementary and widely used approach is the Visual Predictive Check (VPC).

You may come across this in p

The principle is straightforward: if our model is a good description of the data, then concentrations simulated from the model should look like the concentrations we actually observed. To produce a VPC, the fitted model is used to generate prediction intervals, which are then plotted alongside the observed data. A well-fitting model will have the observed data points falling within these intervals.

vpcPlot(fit) +
  theme_bw(base_size = 11) +
  xlab("Time after dose (h)") +
  ylab("Furosemide concentration (mg/L)")

one.cmt <- function() {
  ini({ 
    tcl <- log(0.01) 
    tv <- log(1)
    eta.cl ~ 0.1
    eta.v ~ 0.1
    prop.sd <- 0.1
  })
  model({
    cl <- exp(tcl + eta.cl)
    v <- exp(tv + eta.v)
    linCmt() ~ prop(prop.sd)
  })
}

Making a reference

When we include a continuous covariate such as weight in a population pharmacokinetic model, we need to define a meaningful reference point.

Because clearance is estimated in the exponential domain, the typical population clearance is only recovered when the covariate term equals zero. So we need to ensure that zero corresponds to a clinically sensible value.

By expressing weight as log(WT/median_weight), the covariate term equals zero when an individual has exactly the median weight of the study population, meaning our typical clearance estimate corresponds to a patient of average weight.

Without this step, zero weight would be the reference point, which is biologically meaningless. Centering around the median is therefore standard practice and ensures that the parameter estimates you report are both mathematically sound and clinically interpretable.

mw<- pheno %>% 
  group_by(ID) %>% 
  slice(1) %>% # take only one weight per patient
  ungroup() %>% 
  summarise(median(WT)) %>% 
  as.numeric

We know from our summary statistics (Table 1) that the median weight for individuals in the phenobarbital data set is 1.3 kg, and that this is a set of neonatal data. We should therefore choose 1.3 kg as our reference weight.

Modelling categorical covariates

The easiest example of modelling a categorical covariate is where you have two categories — a common example of this might be smoking/non-smoking. In a dataset, this should be coded with ‘0’ (reference group, e.g. non-smoking) or ‘1’.

The full model for the covariate, including inter-individual variability, is (with smoking as an example):

\[CL_i = \theta_{CL} \times e^{\eta_{CL}} \times (1 + SM \times \theta_{smoke})\]

As \(SM\) is smoker (1=yes, 0=no), you can see that for a non-smoker the covariate term drops out entirely (since \(SM=0\), \(SM \times \theta_{smoke} = 0\)), and individual clearance reduces to:

\[CL_i = \theta_{CL} \times e^{\eta_{CL}}\]

You would all this to your model block like this:

cl <- exp(tcl + eta.cl) * (1 + SM*tsm)

For a smoker (\(SM=1\)), clearance is scaled by \((1 + \theta_{smoke})\), where \(\theta_{smoke}\) represents the proportional difference in clearance between smokers and non-smokers. A positive value of \(\theta_{smoke}\) indicates higher clearance in smokers, and a negative value indicates lower clearance.

Simulations

Building and refining a population pharmacokinetic model is not an end in itself. Once we have a well-fitted model that adequately describes drug exposure and accounts for key sources of variability we can use it for simulations and dose optimisation.

Having established a covariate relationship between sepsis and clearance, we can derive individual predicted clearance estimates for each patient and use these to simulate concentration-time profiles under alternative dosing regimens. In a septic patient with enhanced renal clearance, standard antibiotic dosing may fail to achieve pharmacokinetic/pharmacodynamic targets. This hypothesis can be explored using a simulation from a well-fitted population pharmacokinetic model.

In simulation, the parameter estimates and their associated variability are used to generate a large number of simulated individuals. The proportion of simulated patients achieving the pharmacokinetic/pharmacodynamic target under each dosing scenario can then be compared across regimens to identify the optimal dose and frequency of administration.

This model-informed precision dosing framework is increasingly used in clinical practice for drugs with a narrow therapeutic index, and underpins adaptive dosing strategies in therapeutic drug monitoring programmes. It is also central to drug development, allowing prospective evaluation of dose selection and sampling strategies before committing to large and expensive studies.
