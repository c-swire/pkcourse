Introduction

‘Rich’ data are where there are lots of samples for each individual at each time point. This kind of data is really easy to produce in early-phase trials, including entry-into-human studies.

However, at later phases of drug development, including post-marketing, pharmacokinetic data are often more challenging to obtain. In addition, rigid sampling times are less easy to implement outside of the confines of a dedicated pharmacokinetic study as patients or study participants are often at clinic or study visits at irregular times and so sampling may be somewhat opportunistic. Self-administered doses are also less likely to be accurately taken at prescribed times. These factors make modelling data using the non-compartmental methods we have used so far challenging and less accurate.

The purpose of this session, is to introduce the concept of non-linear mixed-effects modelling. This will include introduction to the modelling package nlmixr. This package is challenging to install on your own machine, install.packages(‘nlmixr2’) may work. If you run a Windows operating system, you are likely to also need rtools. You will need to follow instructions online on how to install and download this. Installation on an Apple Mac usually requires Xcode command line tools to be downloaded. We recommend using Posit Cloud for this session.

Advantages and disadvantages of non-linear mixed-effects models

Advantages of non-linear mixed-effects approach

  • Enables modelling of sparse data, useful in later phase studies
  • Allows more complex identification of covariate effects (for example organ function)
  • Alternative sampling and modelling may allow for populations to be studied that are not straightforward to study in traditional phase-0/1 design (for example children, or drugs in pregnancy)
  • Allows complex simulations which can inform subsequent study design and drug dosing

Disadvantages of non-linear mixed-effects approach

  • Feels inherently more complex which can make people anxious about undertaking or understanding the process
  • Computationally expensive
  • Time consuming
  • Often requires time cleaning/organising data from multiple sources

What is the point of a model?

It is clear that understanding data from a clinical trial is important. Pharmacokinetic data is the same as any other output from a trial. It is important to understand the data, what it means for the absorption, distribution, metabolism and excretion (ADME) of the compound AND it allows us to make suggestions for the next stage of drug development. This might include dose selection, frequency of administration or even decisions on formulation.

Compartmental pharmacokinetic models simplify complex human physiology by describing the concentration-time course of a drug with (relatively) simple mathematical equations. These mathematical equations help to quantify ADME processes through the estimation of values for parameters such as clearance and volume of distribution.

One-compartment IV bolus model

In the one compartment model IV bolus model, the body is simplified and represented as a single box, into which the dose of the drug is deposited instantaneously. Drug is assumed to distribute evenly and instantaneously throughout this compartment such that the contents of the box is assumed to be the same wherever you are within it.

Figure 1: One-compartment model

This single compartment is assumed to receive the drug instantly at the time of the intravenous dose. The compartment has a volume \(V_1\), which in a one-compartment model is equal to the volume of distribution (\(V_d\)) of the drug. The concentration in this compartment is equal to the plasma concentration (\(C_p\)) and the amount of drug in the compartment (\(A_1\)) is equal to the total amount of drug in the body (\(A_b\)). Elimination of the drug is usually assumed to be a first-order process with elimination rate constant \(k_e\).

Relationship between \(k_e\), \(V_d\) and \(Cl\) in the IV bolus model

\(k_e, V_1\) and clearance (\(Cl\)) are related \[\begin{equation} k_e=\frac{Cl}{V_1} \end{equation}\].

Concentration-time equation for the one-compartment IV bolus model

The concentration \(C\) at time \(t\) for the one-compartment IV bolus model is \[\begin{equation} C=C_0e^{-kt} \end{equation}\] or, if you prefer to use clearance and volume of distribution \[\begin{equation} C=C_0e^{-\frac{Cl}{Vd}t} \end{equation}\] \(C_0=\) Drug concentration at time \(t=0\).

Population pharmacokinetic modelling

What is mixed effects modelling?

In a population pharmacokinetic model, we assume that each individual’s pharmacokinetic parameters (e.g. clearance) are made up of two components.

The fixed effect represents the typical value across the population. For example, the typical clearance of Drug A in healthy volunteers. This is what we usually report as the headline result of a population pharmacokinetic analysis.

The random effect represents how much any given individual deviates from that typical value. Not everyone has the same clearance. Some patients clear the drug faster, some slower. The random effect captures this between-subject variability. We cannot predict it perfectly, but we can estimate how large it is.

A mixed-effects model contains both components.

How this applies to compartmental pharmacokinetic modelling

Let’s return to the one-compartment IV-bolus model once more. Let us parameterise using primary pharmacokinetic parameters (clearance and volume of distribution). Broadly, our model remains the same \[C=C_{0}e^{-\frac{Cl}{V}t}\] however, we now add a further layer, by stating that our parameters are made up of a fixed (population mean, \(\theta\)) effect and a random (inter-individual variability, \(\eta\)) effect. \[Cl=\theta_{Cl}\times e^{\eta_{Cl}}\] the modelling process then estimates \[\theta_{Cl}\textrm{ -the population mean value for clearance (a fixed effect)}\] and makes some estimate of \[\eta_{Cl} \textrm{ -individual variability of clearance (a random-effect)}\]

Similarly, we model volume of distribution as \[V=\theta_{V}\times e^{\eta_{V}}\]

Non-linear mixed-effects modelling in R

Non-linear mixed-effects modelling is computationally complex and requires knowledge of statistics, pharmacological models and how to use computer code. We are going to enter into the world of population pharmacokinetic modelling by using the package nlmixr.

Using nlmixr

First you need to call library(nlmixr2) to load the package.

In this example we fit a one-compartment oral-absorption model to sparse data after an oral dose of furosemide is taken.

sparse<-
 read_csv("https://raw.githubusercontent.com/dlonsdal/SGUL_PK_data/main/sparsefru.csv")

nlmixr2 requires data to be structured and labelled in a particular way.

  • ID = patient ID
  • TIME = time (since administration of dose)
  • DV = dependent variable (drug concentration)
  • AMT = amount (dose)
  • EVID = event identifier (tells nlmixr2 whether that row represents an observation or dose event)
  • CMT = compartment (1 = depot compartment, 2 = central compartment).

Then you need to build your model. When building the model, you specify two blocks in a standard R function(). As always, you give your function a name, I have chosen one.cmt.

Ini block

The first part of the function is to set the initial estimates for your model parameters, this is known as the ini() block. We are parameterising using \(k_a\), \(Cl\) and \(V\) in our one-compartment oral-absorption model. So we need to tell nlmixr initial values for each of these parameters as well as values for the random effects associated with each parameter.

Model block

The second part of the function defines the model. The model() block. Here you describe the relationship between the effects.

After building the model we can run a check using nlmixr(one.cmt), which describes the model we have built. And then we fit the model to the data.

sparse<-
 read_csv("https://raw.githubusercontent.com/dlonsdal/SGUL_PK_data/main/sparsefru.csv")


# First make the model
one.cmt <- function() {
  ini({ 
    tka <-log(1.3) # set the initial estimates for each parameter, note that ka will not be required for IV datasets
    tcl <- log(8) 
    tv <- log(20)
    eta.ka~0.1 # inter-individual variabilities
    eta.cl ~ 0.1
    eta.v ~ 0.1
    add.sd <- 0.01 # residual variability
  })
  model({ # define the model
    ka <- exp(tka + eta.ka) # ka will not be required for IV datasets
    cl <- exp(tcl + eta.cl)
    v <- exp(tv + eta.v)
    linCmt() ~ add(add.sd)
  })
}

# Run a check
nlmixr2(one.cmt)

\[\begin{align*} {ka} & = \exp\left({tka}+{eta.ka}\right) \\ {cl} & = \exp\left({tcl}+{eta.cl}\right) \\ {v} & = \exp\left({tv}+{eta.v}\right) \\ linCmt() & \sim add({add.sd}) \end{align*}\]

# Fit the data using this model
fit<-nlmixr2(one.cmt,sparse,est="saem", list(print=0))
## [====|====|====|====|====|====|====|====|====|====] 0:00:00
